# 議事録自動化 セットアップ・運用手順

Teams会議の録画から、議事録（Markdown）を半自動で作成してBacklogに登録するためのツールです。
このドキュメントは **① 初回セットアップ（PCごとに1回）** と **② 普段の議事録作成（毎回）** に分かれています。
**上から順に、コマンドはそのままコピーして実行**すれば、誰がやっても同じ結果になるように書いています。

---

## 前提・用意するもの

- **Windows 10 / 11 のPC**（PowerShellを使用）
- **Backlog の個人APIキー**（各自で発行。②の**会議番号の自動決定（プロンプト生成時）と登録**に使います。発行手順は「① 初回セットアップ A-3」に記載）
- **配布フォルダ一式**（`setup.ps1` などが入ったフォルダ。管理者から受け取る）
- **承認済みAI**（社内で利用が承認されたAIのみ使用する。外部AIの利用は上長承認が前提）

> **言葉の意味**
> - `C:\minutes` … このツールの**実行環境**（セットアップで自動作成される）
> - `C:\minutes\work` … 録画や文字起こし、議事録を置く**作業フォルダ**
> - 録画リンク・登壇者・開催日・会議番号は**すべて自動**で埋まります。毎回入力する項目はありません。

---

# ① 初回セットアップ（PCごとに1回だけ）

## A-1. 配布フォルダを置く

1. 管理者から受け取った**配布フォルダ**を、分かりやすい場所（例：`ドキュメント`）にコピーする。
2. そのフォルダを**エクスプローラーで開く**。
3. エクスプローラー上部の**アドレスバー**をクリックして、`powershell` と入力し **Enter**。
   → そのフォルダの場所でPowerShellが開きます（以降のコマンドはこのウィンドウに貼り付けます）。

## A-2. セットアップを実行する

下のコマンドをそのまま貼り付けて実行します。
（FFmpeg・文字起こしエンジン・モデル約1.5GBのダウンロード、`C:\minutes` の作成、スクリプト配置まで自動で行います。**数分〜十数分**かかることがあります。）

```powershell
powershell -ExecutionPolicy Bypass -File ".\setup.ps1"
```

**完了の目安**：最後に `=== Verify ===` が表示され、次のように**すべて OK**になっていること。

```
  ffmpeg        : OK
  whisper-cli   : OK
  model         : OK (1463 MB)
  transcribe.ps1: OK
  buildprompt.ps1: OK
  register.ps1  : OK
  run.ps1       : OK
  backlog.config: OK ...
```

> `ffmpeg : NOT FOUND` と出た場合は、**PowerShellを一度閉じて新しく開き直し**、もう一度 A-2 を実行してください（PATH反映のため）。

> このコマンドは**繰り返し実行しても安全**です（導入済みのものはスキップ）。**スクリプトを新版に更新したときの再配布**や、既存設定への **`RecFolderUrl` の自動追記**もこの再実行で反映されます（既存の `backlog.config.txt` は上書きされません）。

## A-3. Backlog APIキーを発行して登録する（各自）

APIキーは、**会議番号の自動決定（Backlog参照）** と **議事録の登録** の両方に使います。各自1回設定すれば以降は不要です（未設定だと会議番号はそのPCのローカル連番にフォールバックし、正しい回にならないことがあります）。

**（1）APIキーを発行する**

1. ブラウザで **https://tcs-s.backlog.com** にログイン。
2. 画面右上の**自分のアイコン → 個人設定**。
3. 左メニューの **「API」**。
4. 「メモ」に用途（例：`議事録自動登録`）を入力し、**「登録」**。
5. 表示された**APIキー（英数字の文字列）をコピー**する。
   > このキーは後から再表示できません。確実にコピーしてください。

**（2）APIキーをこのPCに登録する**

下のコマンドの `＜ここにコピーしたAPIキー＞` を置き換えて実行します。

```powershell
setx BACKLOG_API_KEY "＜ここにコピーしたAPIキー＞"
```

→ `成功: 指定した値は保存されました。` と出れば登録完了です。

**（3）反映のためPowerShellを開き直して確認**

`setx` は**新しいウィンドウから有効**になります。**PowerShellを一度閉じて新しく開き**、次で確認します。

```powershell
$env:BACKLOG_API_KEY.Length
```

→ 数字（例：`64`）が表示されればOK。空欄なら（2）をやり直してください。

> **APIキーは各自のもの**です。設定ファイルやフォルダには保存しません（環境変数だけで管理します）。

## A-4. セットアップ完了の最終確認

```powershell
Get-Content "C:\minutes\backlog.config.txt"
```

次の4項目が入っていればOK（これらは**共有の設定で変更不要**）。

```
Space=tcs-s.backlog.com
ProjectId=520525
ParentId=019f3bd7c41e77b78709ccf2885a1175
RecFolderUrl=https://tcs-s.cybozu.com/o/ag.cgi?page=FileIndex&fCID=45883
```

これで初回セットアップは完了です。以降は **② 普段の議事録作成** だけを行います。

# ② 普段の議事録作成（毎回の手順）

## B-1. 録画（.mp4）をダウンロードする

1. サイボウズの録画フォルダを開く：
   **https://tcs-s.cybozu.com/o/ag.cgi?page=FileIndex&fCID=45883**
2. 対象の録画（`.mp4`）を**PCにダウンロード**する（通常は「ダウンロード」フォルダに保存されます）。

> 社内情報のため、外部同期される場所（個人のクラウド共有など）には置かないでください。
> このとき**他の新しい `.mp4` をダウンロードしない**でください（次の手順で「最新の1本」を自動で取り込みます）。

## B-2. 文字起こし〜AIプロンプト生成（1コマンド）

```powershell
powershell -ExecutionPolicy Bypass -File "C:\minutes\run.ps1" -Fetch
```

このコマンドが自動で行うこと：

1. **ダウンロードフォルダの最新 `.mp4` を `C:\minutes\work` へ移動**（`Fetched: ... -> work` と表示）
2. 音声抽出 → **文字起こし**（whisperで**録画時間の約1.5〜2倍**かかります。**ウィンドウは閉じずに放置**）
3. AIに貼り付ける**完成プロンプト**を生成し、`C:\minutes\work\ai_prompt.txt` に保存＋**クリップボードにコピー**。**文字起こし完了後、このプロンプトが自動で開きます**。

> **録画リンク・登壇者・開催日・会議番号はすべて自動**で入ります（入力不要）。
> - 録画リンク＝共有フォルダURL（設定の `RecFolderUrl`）
> - 登壇者＝空欄のままAIが文字起こしから推定（確定できない名前には「（要確認）」が付きます）
> - 開催日＝**録画ファイル名の会議日**（Teams録画名に含まれる `YYYYMMDD`）から自動
> - 会議番号＝**Backlogの登録先（親ドキュメント）の既存議事録から自動決定**（`第N回` の最大＋1）。全員・全PCが同じ親を見るので、**別の人が初めて使っても正しい番号**になります（前の回が登録済みであること、APIキーが設定済み〔A-3〕であることが前提。取得できない場合はそのPCのローカル連番にフォールバックし `[NOTE]` を表示）。
>
> `-Fetch` はダウンロードフォルダに `.mp4` が無いと、**古い録画を誤って処理しないよう停止**します。その場合は B-1 をやり直してください。

## B-3. AIで議事録案を作成する

1. **承認済みAI**を開く。
2. **貼り付け（Ctrl+V）**する（B-2でクリップボードにコピー済み）。そのまま送信。
3. AIが出力した議事録を **`.md` ファイルとしてダウンロード**する（＝ダウンロードフォルダに保存される）。

> プロンプトには「ダウンロードできるMarkdown（.md）ファイルで出力してください」という指示が含まれているので、AIは `.md` のダウンロードを提示します。
> このとき、ダウンロードフォルダに**他の新しい `.md` を置かない**でください（次の手順で最新1本を取り込みます）。無関係な `.md` があれば別フォルダへ退避しておきます。

## B-4. 公開前チェック（人の目で確認・必須）

ダウンロードした議事録（`.md`）を開き、公開してよい内容か確認・修正します。特に次を重点的に：

- **人名・会社名・数字・日付・金額・固有名詞**の誤り（文字起こしの誤認識に注意）
- **機微な内容**（人事・評価・労務・顧客名・未公開案件・契約金額など）が本文に断定で書かれていないか
- 「登壇者」が空や「（要確認）」の場合、必要に応じて正式名に修正

## B-5. Backlogへ登録する

下のコマンド1つで、**ダウンロードフォルダの最新 `.md` を取り込み → Backlog登録**まで行います。

まずAPIキーが今のセッションに入っているか確認（空なら読み込み直す。通常は不要）：
```powershell
if (-not $env:BACKLOG_API_KEY) { $env:BACKLOG_API_KEY = [Environment]::GetEnvironmentVariable("BACKLOG_API_KEY","User") }
```

登録：
```powershell
powershell -ExecutionPolicy Bypass -File "C:\minutes\run.ps1" -To register -Fetch
```

動作の流れ：

1. `Fetched: xxx.md -> work\minutes.md`（最新`.md`の取り込み成功）
2. **タイトル抽出・重複チェック**を表示。`[DUPLICATE]` が出た場合は同名の議事録が既にあるので、内容・タイトルを確認。
3. 確認メッセージ `Register to ... Type y to proceed` が出たら、内容を確認して **`y`** を入力し Enter。`Registered ... Document id ...` と出れば登録完了。

> ⚠️ **`Fetched:` のファイル名と `Title:` が「今回の会議の議事録」か必ず確認**してから `y` を押してください。別の `.md` を拾っていた場合は `y` 以外を入力して中止し、B-3 からやり直します（このアカウントは自分では削除できないため、誤登録は取り消せません）。

> このアカウントは**追加はできても自分では削除できません**（削除は管理者権限）。**重複チェックは `y` を押す前に表示される**ので、そこを最終確認にしてください。

> **AIが `.md` ダウンロードに対応していない（テキストのコピーのみ）場合**は、出力を `C:\minutes\work\minutes.md` に保存し、次で登録します：
> ```powershell
> powershell -ExecutionPolicy Bypass -File "C:\minutes\register.ps1" -DryRun   # 確認（送信しない）
> powershell -ExecutionPolicy Bypass -File "C:\minutes\register.ps1"           # 本登録（yで確定）
> ```

# ③ 設定値（共有・原則変更しない）

`C:\minutes\backlog.config.txt` に入っている値です。**チーム共通**で、通常は触りません。

| キー | 意味 | 値 |
|---|---|---|
| `Space` | Backlogのスペース | `tcs-s.backlog.com` |
| `ProjectId` | 登録先プロジェクト(ALL)のID | `520525` |
| `ParentId` | 親ドキュメントのID | `019f3bd7c41e77b78709ccf2885a1175` |
| `RecFolderUrl` | 録画リンクに使う共有フォルダURL | `https://tcs-s.cybozu.com/o/ag.cgi?page=FileIndex&fCID=45883` |

- **APIキーはここに書きません**（各自の環境変数 `BACKLOG_API_KEY` で管理）。
- 録画の保存フォルダが変わったときだけ、`RecFolderUrl` を新しいURLに書き換えます。

# ④ 困ったとき（トラブルシュート）

| 症状 | 対処 |
|---|---|
| `ffmpeg ... NOT FOUND` | PowerShellを閉じて**新しく開き直し**、`setup.ps1` を再実行 |
| `$env:BACKLOG_API_KEY.Length` が空 | `setx ...` の後は**新しいPowerShell**で確認。未登録なら A-3(2) をやり直す |
| `-Fetch failed - stopping...` | ダウンロードフォルダに対象ファイルが無い/移動できない。文字起こし時は`.mp4`（B-1）、登録時は`.md`（B-3）を落とし直す |
| `[WARN] Link is blank ...` | 設定に `RecFolderUrl` が無い。`setup.ps1` を再実行すると自動追記される |
| `[DUPLICATE] ...`（登録時） | 同名の議事録が既にある。タイトル・内容を確認 |
| 文字化けして見える | ファイルはUTF-8で正しい。読むときは `Get-Content ファイル -Encoding UTF8` を付ける |
| 会議中の音声が入っていない | 録画に音声トラックが無い可能性。取得元・録画設定を確認 |

## 補足：作業の全体像

```
録画(.mp4)をダウンロード
   │  run.ps1 -Fetch
   ▼
文字起こし → AIプロンプト生成（クリップボードへコピー）
   │  承認済みAIに貼付・送信
   ▼
AI出力を .md でダウンロード
   │  公開前チェック（人・必須）
   ▼
run.ps1 -To register -Fetch  ─▶  Backlogに議事録
```

- **自動**：録画/議事録の取り込み・音声抽出・文字起こし・プロンプト生成・会議情報の埋め込み・Backlog登録
- **人が行う**：AIへの貼り付けと `.md` ダウンロード、公開前チェック（この2つは必ず人が実施）
