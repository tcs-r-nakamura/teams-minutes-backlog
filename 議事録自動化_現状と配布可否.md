# 議事録自動化ツール：現状まとめ・配布可否チェック

作成日：2026-07-10

## 0. 概要

Teams会議の録画から議事録の草案を半自動で作成し、Backlog へ登録するツールです。
文字起こしは社内PC内で完結し（外部送信なし）、草案作成のみ承認済みAIを利用します。録画をダウンロードした後は、実質2コマンドで運用できます。

---

## 1. 自動化できていること

1. 録画の取り込み（ダウンロードフォルダの最新mp4 → 作業フォルダへ自動移動）
2. 音声抽出（FFmpeg）
3. 文字起こし（whisper.cpp、社内PC内で完結・外部送信なし）
4. 会議情報の自動決定
   - 会議番号：Backlog にある既存議事録から自動採番（第N回＋1／全員・全PCで一致）
   - 開催日：録画ファイル名に含まれる会議日時から取得
   - 録画リンク：サイボウズの共有フォルダを既定でセット（個別URLの指定も可）
   - 登壇者：AIが文字起こしから推測（名前の指定も可）
5. AIに渡すプロンプトの自動生成（クリップボードへコピー＋自動表示）
6. Backlog登録（同名チェック＋登録前の内容確認）

## 2. できていないこと（人手 or 未対応）

- AIへの送信・草案の受け取り：現状は人がコピペし、出力の .md をダウンロード（承認済みAIをブラウザで利用しているため）
- 公開前チェック（人名・数値・日付・機微情報）：品質・コンプライアンス上、意図的に人手を残す工程
- サイボウズからの録画ダウンロード：公式APIが無く手動（録画破損時は Teams から取得）
- 登壇者の最終確定：AI推測は誤り得るため人が確認

## 3. 今後の方針

- さくらのAI Engine（小見山さん承認済・国内完結でセキュア・無料枠で各種モデル＝Whisper含む）を活用予定
  - AIへの送信をAPI化 → 「文字起こし → 草案 →（人手チェック）→ 登録」をさらに一気通貫へ
  - 文字起こし（Whisper）をGPU側にオフロード → 高速化・低スペックPCでも実行可
  - ※サービス未登録のため即時利用は不可。登録・整備でき次第、順次連携
- 公開前チェックは引き続き人手を維持する方針

## 4. 依存関係・配布可否（共有前チェック：確認済み）

### 依存しているもの
- Windows 10 / 11 ＋ PowerShell 5.1（OS標準）
- FFmpeg … セットアップが静的ビルドを GitHub から直接ダウンロード（winget・管理者権限は不要）
- 文字起こしエンジン whisper.cpp（v1.9.1、GitHub から自動ダウンロード）
- 文字起こしモデル ggml-medium.bin（約1.5GB、Hugging Face から自動ダウンロード）
- Backlog API（ネットワーク）＋ 各自の APIキー（環境変数で管理）
- 承認済みAI（現状は人がブラウザで利用）

### 「配布するだけで使えるか」
結論：使えます。ZIPを展開 → セットアップ実行 → 各自APIキー設定、で稼働します（配布物は全ファイル・依存の充足を監査済み）。FFmpeg も直接ダウンロードに変更したため、winget や管理者権限は不要になり、制限の厳しいPCでも通りやすくなりました。

初回導入に必要な前提は、実質「インターネット接続」のみです：
- `github.com`（文字起こしエンジン・FFmpeg）／ `huggingface.co`（モデル約1.5GB）に到達できること（プロキシ／ファイアウォールで遮断されていないこと）

補足：
- 依存物（FFmpeg・エンジン・モデル）を外部から取得するため、完全オフライン配布ではありません（将来、社内ミラー同梱で緩和可能）。
- whisper.cpp は v1.9.1 に固定。FFmpeg は静的ビルドを直接取得（単体実行可、DLL不要）。
- ウイルス対策ソフト（例：ウイルスバスター）が、DLした未署名の実行ファイル（`whisper-cli.exe`／`ffmpeg.exe`）を初回に「未知のプログラム」として確認/ブロックすることがあります。警告のファイルが `C:\minutes\tools` 配下（setup が導入した場所）かを確認し、社内ルールに従って許可します。※実機で `whisper-cli.exe` が公式リリースと SHA-256 一致することを確認済み（＝公式配布物と同一バイナリであることの確認であり、マルウェアでないことの保証ではありません）。会社PCで頻発する場合は IT に `C:\minutes\tools` の許可を相談。
- APIキーはファイルに保存せず各自の環境変数で管理。録画・議事録などの社内情報は外部同期される場所に置かない運用です。

### 配布方法
- チーム配布：スターターセット一式を ZIP（`議事録自動化_v1_20260710.zip`）にまとめ、サイボウズに設置。手順書（`セットアップと運用手順.md`）同梱。
- 保守元：GitHub（private）リポジトリで版管理。

## 5. まとめ

- 録画取り込み〜文字起こし〜会議情報の自動決定〜Backlog登録までは自動化済み。実際の録画で登録まで動作確認済み。
- 人が担うのは「AIへの草案生成の橋渡し」と「公開前チェック（意図的に人手）」。
- インターネット（github.com・huggingface.co 到達）さえあれば、配布するだけで利用可能（winget・管理者権限は不要）。
- 次段はさくらのAI Engineで草案生成のAPI化・文字起こしのGPU化を進める。
