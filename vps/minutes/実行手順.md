# minutes（VPS版）実行手順 — Step 2（CLI）

AI-OCR用VPS（NixOS）の `~/projects/minutes/` に雛形を載せて、録画から議事録の
下書きを作るまでの手順です。各コマンドの先頭に **[PC]**（Windows PowerShell）／
**[VPS]**（SSHで入ったサーバー側）を明記します。

## 0. 前提
- PCの `~/.ssh/config` に `Host tcs-ocr`（HostName/User aiocr/Port 2201/IdentityFile）が設定済み。
- さくらAI Engineの APIトークンを発行済み（`<UUID>:<シークレット>` の1本）。
- 雛形は PC の `C:\Users\r-nakamura\source\議事録自動化\vps\minutes\` にある。

---

## マイルストーン1：サーバーに載せて“環境が動く”か確認

### 1-1. [PC] `~/projects` を作成し、既存の上書きを確認
```powershell
ssh tcs-ocr "mkdir -p ~/projects"
ssh tcs-ocr "test -e ~/projects/minutes && echo EXISTS || echo OK"
```
→ OK なら未作成で安全。EXISTS なら転送先を変える（例：minutes2）か中身を確認。

### 1-2. [PC] 雛形フォルダを転送
```powershell
scp -r "C:\Users\r-nakamura\source\議事録自動化\vps\minutes" tcs-ocr:~/projects/
```

### 1-3. [VPS] サーバーに入り、権限を絞って環境を起動
```bash
ssh tcs-ocr
cd ~/projects/minutes
chmod 700 ~/projects/minutes     # 社内情報を他ユーザーから隠す
nix flake lock                   # 依存バージョンを固定（flake.lock作成）
nix develop                      # 初回はPython/ffmpeg取得で数分（止まって見えても正常）
```
- `ssh tcs-ocr` … サーバーに入る
- `cd` … フォルダへ移動
- `chmod 700` … 本人だけアクセス可の権限に
- `nix flake lock` … 使う道具のバージョン固定
- `nix develop` … レシピ通り道具が揃った作業環境に入る（この中だけで有効）

### 1-4. [VPS]（nix develop に入った状態で）道具を確認
```bash
python --version
ffmpeg -version | head -1
python -c "import requests; print('requests ok')"
```
期待：`Python 3.x` ／ `ffmpeg …` ／ `requests ok`

## マイルストーン2：実際に録画から下書きを作る

### 2-1. [PC] 録画と字幕を inbox/ へ転送
`src/main.py` は入力を自動取得しない。実行前に inbox/ へ置く。
ファイル名に日時トークン（YYYYMMDD_HHMMSS）が含まれると開催日を自動判定する。
```powershell
scp "C:\minutes\work\＜録画＞.mp4" tcs-ocr:~/projects/minutes/inbox/rec_20260703_165826.mp4
scp "C:\minutes\work\teams_20260703_165826.vtt" tcs-ocr:~/projects/minutes/inbox/teams.vtt
```

### 2-2. [VPS] トークンを（履歴に残さず）読み込む
```bash
cd ~/projects/minutes
nix develop
read -rs SAKURA_AI_TOKEN; echo; export SAKURA_AI_TOKEN
```
→ 入力は表示されず、値はシェル履歴に残らない（このセッション中だけ有効）。

### 2-3. [VPS] 下書きを生成（必ずプロジェクトフォルダ内で）
```bash
python -m src.main draft inbox/rec_20260703_165826.mp4 --vtt inbox/teams.vtt --no 1
```
- `--no` 会議番号（未指定は「要確認」）。名前にトークンが無ければ `--date 2026/07/03` を付ける。
- 処理：音声抽出 → 30分超なら無音分割 → さくらWhisper文字起こし → 2ソースのプロンプト → さくらAIで下書き。
出力：`out/minutes.md`

### 2-4. [VPS] 出来を確認
```bash
head -20 out/minutes.md
```
- `--no` を省いた場合、会議番号は Backlog から自動採番（親配下の「第N回」最大+1。
  同一日付が既にあればその番号を再利用）。取得できない時は「要確認」のまま。

## マイルストーン3：人がチェックしてから Backlog へ登録

`out/minutes.md` を**人が確認・修正**してから登録する。タイトルは先頭の `# 見出し`。

### 3-1. [VPS] Backlog APIキーを（履歴に残さず）読み込む
```bash
read -rs BACKLOG_API_KEY; echo; export BACKLOG_API_KEY
```

### 3-2. [VPS] まず送らずに確認（dry-run）
```bash
python -m src.main register --dry-run
```
- タイトル・宛先・重複/番号衝突チェックの結果だけ表示（送信しない）。
- `[DUPLICATE]` … 同名文書が既存 → 登録中止。
- `[NUMBER CONFLICT]` … 同じ「第N回」を別文書が使用 → 番号を直す。
- `[ERROR] duplicate check could not run` … Backlog へ到達不可 → **既定で中止**（安全側）。

### 3-3. [VPS] 登録（確認プロンプトあり）
```bash
python -m src.main register            # 実行後 y を入力で送信
```
- `--md <path>` 別ファイルを指定 ／ `--title "..."` タイトル上書き。
- `--force` 確認プロンプトを省略（自動運用向け。重複/番号チェックは維持）。
- `--allow-unchecked` チェックが走らない時でも登録を強行（非常時のみ）。

### 3-4. [確認] Backlog を開いて登録内容を目視確認
- このアカウントは追加はできるが**削除はできない**。誤登録時は管理者へ連絡。

## トラブルシュート
| 症状 | 対処 |
|---|---|
| nix develop が experimental features エラー | `nix --extra-experimental-features 'nix-command flakes' develop` |
| 初回DLが遅い/失敗 | VPSから cache.nixos.org / github.com への到達を確認 |
| SAKURA_AI_TOKEN is not set | 2-2 を実行し直す（nix develop を抜けると環境変数は消える） |
| テンプレート未検出 / 変な work out ができる | プロジェクトフォルダ（~/projects/minutes）で実行しているか確認 |

## 注意
- 音声・文字起こしはさくらAI Engine（承認済み・国内完結・学習利用なし）へ送信される。
- inbox/ work/ out/ の中身は社内情報。外部同期される場所に置かない。
- トークンは環境変数のみ（ファイル・チャット・メールに貼らない）。
