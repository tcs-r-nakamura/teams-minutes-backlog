# 議事録自動化 セットアップ・運用手順

Teams会議の録画から、議事録（Markdown）を半自動で作成してBacklogに登録するためのツールです。
このドキュメントは ① 初回セットアップ（PCごとに1回） と ② 普段の議事録作成（毎回） に分かれています。
上から順に、コマンドはそのままコピーして実行すれば、誰がやっても同じ結果になるように書いています。

---

## 前提・用意するもの

- Windows 10 / 11 のPC（PowerShellを使用）
- `setup.ps1` を実行できる環境（初回導入に必要）:
  - インターネット接続（初回に github.com〔文字起こしエンジン・FFmpeg〕と huggingface.co〔モデル約1.5GB〕からダウンロード。プロキシ/ファイアウォールでこれらが塞がれていると失敗します）
  - ※ FFmpeg も直接ダウンロードするため、winget や管理者権限は不要です（すべて `C:\minutes` 配下に展開されます）
- Backlog の個人APIキー（各自で発行。②の会議番号の自動決定（プロンプト生成時）と登録に使います。発行手順は「① 初回セットアップ A-3」に記載）
- 配布フォルダ一式（`setup.ps1` などが入ったフォルダ。管理者から受け取る）
- 承認済みAI（社内で利用が承認されたAIのみ使用する。外部AIの利用は上長承認が前提）

> 言葉の意味
> - `C:\minutes` … このツールの実行環境（セットアップで自動作成される）
> - `C:\minutes\work` … 録画や文字起こし、議事録を置く作業フォルダ
> - 開催日・会議番号は自動で決まります。録画リンク・登壇者は必要なら記入でき、書かなければ既定（親フォルダのURL／AIの推測）が入ります。

# ① 初回セットアップ（PCごとに1回だけ）

## A-1. 配布フォルダを置く

1. 管理者から受け取った配布フォルダを、分かりやすい場所（例：`ドキュメント`）にコピーする。
2. そのフォルダをエクスプローラーで開く。
3. エクスプローラー上部のアドレスバーをクリックして、`powershell` と入力し Enter。
   → そのフォルダの場所でPowerShellが開きます（以降のコマンドはこのウィンドウに貼り付けます）。

## A-2. セットアップを実行する

下のコマンドをそのまま貼り付けて実行します。
（FFmpeg・文字起こしエンジン・モデル約1.5GBのダウンロード、`C:\minutes` の作成、スクリプト配置まで自動で行います。数分〜十数分かかることがあります。）

```powershell
powershell -ExecutionPolicy Bypass -File ".\setup.ps1"
```

補足：`.\setup.ps1` の `.\` は「今いるフォルダにある setup.ps1」という意味です。A-1 のとおり、配布フォルダの中で開いた PowerShell で実行してください。別の場所（例：`PS C:\>`）で開いてしまい「`-File` パラメーターの引数 '.\setup.ps1' は存在しません」と出た場合は、先に配布フォルダへ移動してから上のコマンドを実行します：

```powershell
cd "＜配布フォルダのパス＞"
```

（プロンプトが `PS ...\配布フォルダ>` のように配布フォルダを指していればOK。フルパスで `-File "＜配布フォルダのパス＞\setup.ps1"` と直接指定してもかまいません）

完了の目安：最後に `=== Verify ===` が表示され、次のようにすべて OKになっていること。

```
  ffmpeg        : OK
  whisper-cli   : OK
  model         : OK (1463 MB)
  transcribe.ps1: OK
  buildprompt.ps1: OK
  register.ps1  : OK
  run.ps1       : OK
  backlog.config: OK ...
```

> `ffmpeg : NOT FOUND` と出た場合は、FFmpegのダウンロード/展開に失敗しています。`github.com` に到達できるか確認し、もう一度 A-2 を実行してください（できている物はスキップされます）。

> このコマンドは繰り返し実行しても安全です（導入済みのものはスキップ）。スクリプトを新版に更新したときの再配布や、既存設定への `RecFolderUrl` の自動追記もこの再実行で反映されます（既存の `backlog.config.txt` は上書きされません）。

## A-3. Backlog APIキーを発行して登録する（各自）

APIキーは、会議番号の自動決定（Backlog参照） と 議事録の登録 の両方に使います。各自1回設定すれば以降は不要です（未設定だと会議番号はそのPCのローカル連番にフォールバックし、正しい回にならないことがあります）。

（1）APIキーを発行する

1. ブラウザで https://tcs-s.backlog.com にログイン。
2. 画面右上の自分のアイコン → 個人設定。
3. 左メニューの 「API」。
4. 「メモ」に用途（例：`議事録自動登録`）を入力し、「登録」。
5. 表示されたAPIキー（英数字の文字列）をコピーする。
   > このキーは後から再表示できません。確実にコピーしてください。

（2）APIキーをこのPCに登録する

下のコマンドの `＜ここにコピーしたAPIキー＞` を置き換えて実行します。

```powershell
setx BACKLOG_API_KEY "＜ここにコピーしたAPIキー＞"
```

→ `成功: 指定した値は保存されました。` と出れば登録完了です。

（3）反映のためPowerShellを開き直して確認

`setx` は新しいウィンドウから有効になります。PowerShellを一度閉じて新しく開き、次で確認します。

```powershell
$env:BACKLOG_API_KEY.Length
```

→ 数字（例：`64`）が表示されればOK。空欄なら（2）をやり直してください。

> APIキーは各自のものです。設定ファイルやフォルダには保存しません（環境変数だけで管理します）。

## A-3b. （任意）さくらAI EngineのAPIトークンを登録する

AIによる議事録下書きを自動生成する（B-2の自動化・ChatGPT手貼りを省略）場合だけ設定します。使わない場合（ChatGPTに手で貼る運用）は不要です。

> 注意：トークンの配布は当面 管理者・限定利用に留めます（全社展開はサーバー版で行う予定のため）。

1. さくらのコントロールパネルでAPIトークンを発行する（`<UUID>:<シークレット>` の形式の1本の文字列）。
2. このPCに登録する（`＜…＞` を発行した実際のトークンに置き換え）：

```powershell
setx SAKURA_AI_TOKEN "＜発行したトークン全体（コロン込み）＞"
```

3. PowerShellを開き直して確認：

```powershell
$env:SAKURA_AI_TOKEN.Length
```

→ 数字が表示されればOK。

> トークンは秘密情報です。ファイルやチャット・メールに貼らず、環境変数だけで管理します（各自のもの）。

## A-4. セットアップ完了の最終確認

```powershell
Get-Content "C:\minutes\backlog.config.txt"
```

次の4項目が入っていればOK（これらは共有の設定で変更不要）。

```
Space=tcs-s.backlog.com
ProjectId=520525
ParentId=019f3bd7c41e77b78709ccf2885a1175
RecFolderUrl=https://tcs-s.cybozu.com/o/ag.cgi?page=FileIndex&fCID=45883
```

これで初回セットアップは完了です。以降は ② 普段の議事録作成 だけを行います。

# ② 普段の議事録作成（毎回の手順）

## B-1. 録画（.mp4）とトランスクリプト（.vtt）をダウンロードする

1. サイボウズの録画フォルダを開く：
   https://tcs-s.cybozu.com/o/ag.cgi?page=FileIndex&fCID=45883
2. 対象の録画（`.mp4`）をPCにダウンロードする（通常は「ダウンロード」フォルダに保存されます）。
3. 同じ会議の トランスクリプト（`.vtt`。ファイル名末尾が `会議のトランスクリプト.vtt`）も、あればダウンロードする（同じダウンロードフォルダへ）。

> `.vtt` は Teams が作る発言者名つきの字幕記録です。あると 登壇者名が自動で埋まり、文字起こしの精度も上がります（whisperの結果と突き合わせて議事録が作られます）。無くても動きます（その場合はwhisperのみ）。
> `.mp4` と `.vtt` は同じ会議のもの（ファイル名の日時 `YYYYMMDD_HHMMSS` が一致するもの）を落としてください。日時で自動照合するので、取り違えても別会議の `.vtt` は使われません。

> 社内情報のため、外部同期される場所（個人のクラウド共有など）には置かないでください。
> このとき他の新しい `.mp4` / `.vtt` をダウンロードしないでください（次の手順で「最新の1本」を自動で取り込みます）。

## B-2. 文字起こし〜下書きの用意（1コマンド）

やり方は2通りあります。**さくらAI Engineのトークンを設定済み〔A-3b〕なら「方法①（自動）」がおすすめ**です。

### 方法①（AI下書きまで自動・要トークン）
```powershell
powershell -ExecutionPolicy Bypass -File "C:\minutes\run.ps1" -To draft -Fetch
```
- 文字起こし → **さくらAI Engine（gpt-oss-120b）でAI下書きを自動生成** → `C:\minutes\work\minutes.md` に保存、まで一気に行います。
- **B-3（ChatGPT手貼り）は不要**。そのまま B-4（公開前チェック）へ進みます。
- 使用AIは承認済み・国内完結。文字起こしはこれまで通りPC内で行います。

### 方法②（AIは手動・ChatGPTに貼る／トークン不要）
```powershell
powershell -ExecutionPolicy Bypass -File "C:\minutes\run.ps1" -Fetch
```
- 文字起こし → AIに貼るプロンプトを生成して自動で開きます。この後は B-3 に進みます。

以下は両方に共通の「このコマンドが自動で行うこと」：

1. ダウンロードフォルダの最新 `.mp4` を `C:\minutes\work` へ移動（`Fetched: ... -> work` と表示）。同じ会議の `.vtt` があれば一緒に取り込みます（`Fetched: ...vtt -> work\teams_...vtt`）。無ければ `[NOTE] No matching Teams .vtt ...` と出てwhisperのみで進みます。
2. 音声抽出 → 文字起こし（whisperで録画時間の約1.5〜2倍かかります。ウィンドウは閉じずに放置）。`.vtt` があれば、その発言者名つきテキストも作られます（`[Teams] transcript parsed: N speakers ...` と表示）。
3. AIに貼り付ける完成プロンプトを生成し、`C:\minutes\work\ai_prompt.txt` に保存＋クリップボードにコピー。文字起こし完了後、このプロンプトが自動で開きます。`.vtt` があれば、プロンプトにTeamsトランスクリプトとwhisperの両方が入り、登壇者欄がTeamsの発言者名で自動で埋まります。

> ⚠️ 初回のここ（文字起こしの開始時）で、ウイルス対策ソフトが `ffmpeg.exe` / `ffprobe.exe` / `whisper-cli.exe` を「未知のプログラム」として確認/ブロックすることがあります（`setup.ps1` がDLした未署名の実行ファイルのため。**初回だけ**）。
> - 表示されたら「許可」してください（処理が進みます）。放置するとタイムアウトで自動ブロックされ、文字起こしが失敗します。
> - 「なかなか進まない」ときは、この確認ダイアログが裏に出ていないか確認してください。
> - 詳しくは「④ 困ったとき」を参照。会社PCで頻発する場合は IT部門に `C:\minutes\tools` の許可を相談。

> 開いたプロンプト（`ai_prompt.txt`）の「録画リンク」と「登壇者」は、送信前に必要なら記入できます（任意）。
> - 録画リンク：対象の録画をサイボウズで開いたときのファイルURLを貼っておくと、そのリンクが議事録に入ります。貼らなければ、録画が入っている親フォルダのURL（設定の `RecFolderUrl`）が自動で入ります。
> - 登壇者：`.vtt` があれば Teamsの発言者名が自動で入ります（そのままでOK。手で書けば上書きできます）。`.vtt` が無いときは、書いておけばその名前が使われ、書かなければAIが文字起こしから推測します（確定できない名前には「（要確認）」が付きます）。
> - 発言者名の表記（姓名の間の半角スペースなど）を直したいときは、`C:\minutes\speaker_aliases.txt` に `元の名前 = 表示したい名前` を1行足します（例：`中村流星 = 中村 流星`）。
> - 記入したら、そのファイルで Ctrl+A → Ctrl+C で全選択コピーし直してから貼り付けてください（自動コピーは記入前の内容のため）。
>
> 開催日・会議番号は自動で決まるので入力は不要です。
> - 開催日＝録画ファイル名の会議日（Teams録画名に含まれる `YYYYMMDD`）から自動
> - 会議番号＝Backlogの登録先（親ドキュメント）の既存議事録から自動決定（「第N回」の最大＋1）。全員・全PCが同じ親を見るので、別の人が初めて使っても正しい番号になります（前の回が登録済み・APIキー設定済み〔A-3〕が前提。取得できないときはそのPCのローカル連番にフォールバックし `[NOTE]` を表示）。
>
> `-Fetch` はダウンロードフォルダに `.mp4` が無いと、古い録画を誤って処理しないよう停止します。その場合は B-1 をやり直してください。

## B-3. AIで議事録案を作成する（方法②を選んだ場合のみ）

> B-2で「方法①（`-To draft`）」を使った場合は、下書き（`work\minutes.md`）が既にできているので、このB-3は飛ばして B-4 へ進みます。

1. ChatGPT を開く（モデル：5.5／応答速度：高い を選択）。
2. 開いたプロンプト（`ai_prompt.txt`）を確認し、必要なら「録画リンク」「登壇者」を記入する（B-2 の説明を参照）。記入したら、そのファイルで Ctrl+A → Ctrl+C で全選択コピーし直す（記入しないならクリップボードのままでよい）。
3. ChatGPT に貼り付け（Ctrl+V）して送信する。
4. AIが出力した議事録を `.md` ファイルとしてダウンロードする（＝ダウンロードフォルダに保存される）。

> プロンプトには「ダウンロードできるMarkdown（.md）ファイルで出力してください」という指示が含まれているので、AIは `.md` のダウンロードを提示します。
> このとき、ダウンロードフォルダに他の新しい `.md` を置かないでください（次の手順で最新1本を取り込みます）。無関係な `.md` があれば別フォルダへ退避しておきます。

## B-4. 公開前チェック（人の目で確認・必須）

議事録（`.md`）を開き、公開してよい内容か確認・修正します。ファイルの場所は方法によって異なります：
- 方法①（`-To draft`）：`C:\minutes\work\minutes.md`
- 方法②（ChatGPT手貼り）：ダウンロードフォルダに保存した `.md`

特に次を重点的に：

- 人名・会社名・数字・日付・金額・固有名詞の誤り（文字起こしの誤認識に注意）
- 機微な内容（人事・評価・労務・顧客名・未公開案件・契約金額など）が本文に断定で書かれていないか
- 「登壇者」が空や「（要確認）」の場合、必要に応じて正式名に修正

## B-5. Backlogへ登録する

まずAPIキーが今のセッションに入っているか確認（空なら読み込み直す。通常は不要）：
```powershell
if (-not $env:BACKLOG_API_KEY) { $env:BACKLOG_API_KEY = [Environment]::GetEnvironmentVariable("BACKLOG_API_KEY","User") }
```

登録コマンドは方法によって使い分けます：

- 方法①（`-To draft` で `work\minutes.md` が既にある）→ `-Fetch` を付けない：
```powershell
powershell -ExecutionPolicy Bypass -File "C:\minutes\run.ps1" -To register
```
- 方法②（ChatGPTから `.md` をダウンロードした）→ 最新 `.md` を取り込む `-Fetch` 付き：
```powershell
powershell -ExecutionPolicy Bypass -File "C:\minutes\run.ps1" -To register -Fetch
```

動作の流れ：

1. （`-Fetch`時）`Fetched: xxx.md -> work\minutes.md`（最新`.md`の取り込み成功）
2. タイトル抽出・重複チェックを表示。`[DUPLICATE]` が出た場合は同名の議事録が既にあるので、内容・タイトルを確認。
3. 確認メッセージ `Register to ... Type y to proceed` が出たら、内容を確認して `y` を入力し Enter。`Registered ... Document id ...` と出れば登録完了。

> ⚠️ `Title:` が「今回の会議の議事録」か必ず確認してから `y` を押してください。違う場合は `y` 以外を入力して中止（このアカウントは自分では削除できないため、誤登録は取り消せません）。

> このアカウントは追加はできても自分では削除できません（削除は管理者権限）。重複チェックは `y` を押す前に表示されるので、そこを最終確認にしてください。

> AIが `.md` ダウンロードに対応していない（テキストのコピーのみ）場合は、出力を `C:\minutes\work\minutes.md` に保存し、次で登録します：
> ```powershell
> powershell -ExecutionPolicy Bypass -File "C:\minutes\register.ps1" -DryRun   # 確認（送信しない）
> powershell -ExecutionPolicy Bypass -File "C:\minutes\register.ps1"           # 本登録（yで確定）
> ```

`C:\minutes\backlog.config.txt` に入っている値です。チーム共通で、通常は触りません。

|---|---|---|

# ④ 困ったとき（トラブルシュート）

| 症状 | 対処 |
|---|---|
| モデル/エンジン/FFmpeg のダウンロードが失敗 | プロキシ/FWで github.com・huggingface.co が通るか確認 → `setup.ps1` を再実行（できている物はスキップ、失敗分だけ再取得） |
| `ffmpeg : NOT FOUND`（Verify） | FFmpegのDL/展開に失敗。github.com への到達を確認して `setup.ps1` を再実行 |
| `$env:BACKLOG_API_KEY.Length` が空 | `setx ...` の後は新しいPowerShellで確認。未登録なら A-3(2) をやり直す |
| `-Fetch failed - stopping...` | ダウンロードフォルダに対象ファイルが無い/移動できない。文字起こし時は`.mp4`（B-1）、登録時は`.md`（B-3）を落とし直す |
| `[WARN] Link is blank ...` | 設定に `RecFolderUrl` が無い。`setup.ps1` を再実行すると自動追記される |
| `[ERROR] SAKURA_AI_TOKEN is not set`（方法①） | AI下書き自動化にはトークンが必要。A-3b で `setx SAKURA_AI_TOKEN "..."` → PowerShell開き直し。使わないなら方法②（ChatGPT手貼り）を使う |
| AIの下書きが文字化けする | 最新の `draft.ps1` を使っているか確認（`setup.ps1` 再実行で更新）。旧版はPS5.1の応答文字化けの不具合あり |
| `[DUPLICATE] ...`（登録時） | 同名の議事録が既にある。タイトル・内容を確認 |
| ウイルス対策ソフトが `whisper-cli.exe` / `ffmpeg.exe` を「未知のプログラム」として警告/ブロック | `setup.ps1` がDLした未署名の実行ファイルのため初回に検出されます。警告のファイルが `C:\minutes\tools` 配下（setup が導入した場所）であることを確認し、社内ルールに従って「許可」してください。会社PCで頻発する場合は IT部門に `C:\minutes\tools` 配下の実行許可（ホワイトリスト登録）を相談 |
| 登壇者が空／`.vtt` を落としたのに反映されない | `.mp4` と `.vtt` の日時（`YYYYMMDD_HHMMSS`）が一致しているか確認。B-2 実行時に `[Teams] transcript parsed ...` が出ていれば取り込めています。出ていなければ `.vtt` をダウンロードし直して B-2 からやり直し |
| 発言者名の表記を直したい（姓名スペース等） | `C:\minutes\speaker_aliases.txt` に `元の名前 = 表示したい名前` を追記（例：`中村流星 = 中村 流星`）。姓名の間の全角スペースは書かなくても自動で半角になります |
| 文字化けして見える | ファイルはUTF-8で正しい。読むときは `Get-Content ファイル -Encoding UTF8` を付ける |
| 会議中の音声が入っていない | 録画に音声トラックが無い可能性。取得元・録画設定を確認 |

方法①（AI自動・要トークン）：

録画(.mp4)＋トランスクリプト(.vtt) をダウンロード

   ▼
run.ps1 -To register  ─▶  Backlogに議事録

方法②（AIは手動・ChatGPT）：
```
録画(.mp4)＋トランスクリプト(.vtt) をダウンロード
   │  run.ps1 -Fetch
   ▼

```
