# 環境構築手順書（議事録作成の準備）

議事録作成の**手作業運用**に必要なツールと作業場所を、**他の人でも再現できる形**で用意するための手順。`議事録作成手順書.md` の「0. 事前準備」を実際に行うための詳細版。実機で動作確認済みの内容を反映している。

- 対象OS：Windows 10 / 11（PowerShell）
- 使うツール：FFmpeg（音声抽出）、whisper.cpp（文字起こし）
- 実施記録：実行結果は `手順動作確認.md` に記録する

---

## 再現性の3原則

1. **ユーザー名に依存しないパスを使う**：`C:\Users\<名前>\…` は他人の環境に存在しないため使わない。全員が同じ基点パスを使う。
2. **ツール・作業データのパスはASCII（英数字）にする**：whisper.cpp はC++製で、**日本語（非ASCII）を含むパスのファイルを開けないことがある**（実機テストで、基点を `C:\議事録自動化` にすると whisper 実行時にパスが文字化けしたため、`C:\minutes` に変更した）。
3. **作業データ（社内情報）は共有・クラウド同期しない**：OneDrive配下（デスクトップ／ドキュメント）は自動同期の恐れがあるため避ける。

| 種類 | 中身 | 置き場所 | 共有 |
|---|---|---|---|
| 文書 | 定義書・手順書・確認シート | 版管理（Backlog Git／共有フォルダ等） | する |
| ツール | whisper.cpp本体＋モデル | 各PCの `C:\minutes\tools\whisper` | しない（各自導入） |
| 作業データ | 録画・WAV・文字起こし・議事録 | 各PCの `C:\minutes\work` | しない（社内情報） |

※ FFmpeg は winget導入でPATHに入るため、置き場所を意識しなくてよい。

---

## 0. 前提の確認

この環境で確認済み（`winget` / `curl.exe` が利用可）。

```powershell
winget --version   # 例: v1.29.280
curl.exe --version # 例: curl 8.19.0（PowerShellでは curl でなく curl.exe を使う）
```

---

## 1. 作業場所を用意する

基点フォルダを1行で定義し、その下にサブフォルダを作る。置き場所を変えたい人は `$base` の値だけ変更する（ただしASCIIパスにすること）。

```powershell
$base = "C:\minutes"
New-Item -ItemType Directory -Force "$base\tools\whisper\models" | Out-Null
New-Item -ItemType Directory -Force "$base\work" | Out-Null
Test-Path "$base\tools\whisper\models"; Test-Path "$base\work"   # 両方 True で成功
```

作成後の構成：

```
C:\minutes\
  tools\whisper\           … whisper.cpp 本体（展開先）
  tools\whisper\Release\   … whisper-cli.exe（実行ファイル）
  tools\whisper\models\    … モデル（ggml-medium.bin）
  work\                    … 録画・WAV・文字起こし・議事録（社内情報／非共有）
```

## 2. FFmpeg を導入する（音声抽出に使用）

**導入**：
```powershell
winget install --id Gyan.FFmpeg --source winget
```

**確認**：インストール直後は現在のPowerShellにPATHが未反映のため、**新しいPowerShellを開く**か、下記でPATHを取り込み直してから確認する。
```powershell
$env:Path = [System.Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path","User")
ffmpeg -version
ffprobe -version
```
→ `ffmpeg version 8.1.2-…` / `ffprobe version …` が表示されれば成功（実機で確認済み：Gyan.FFmpeg 8.1.2）。

**根拠**：
- FFmpeg は `音声テキスト化方式選定.md` で音声抽出（16kHz・モノラルWAV）に採用済み。
- `Gyan.FFmpeg` は FFmpeg公式（ffmpeg.org）がWindows向けに案内する主要ビルド。winget に `Gyan.FFmpeg 8.1.2` を確認。
- `--source winget` は、`msstore` ソースの利用規約同意を回避するため（`winget list/search` で同意を求められて失敗した実績が根拠）。

**うまくいかない場合**：
- `ffmpeg` が「認識されない」→ 上のPATH取り込み直し、または新しいPowerShellを開く。
- `winget` が使えない → `choco install ffmpeg`（choco 導入済み）。

## 3. whisper.cpp を導入する（文字起こしに使用）

### 3.1 本体（Windows x64 CPU版）を取得・展開

```powershell
$base = "C:\minutes"
curl.exe -L -o "$base\tools\whisper\whisper-bin-x64.zip" "https://github.com/ggml-org/whisper.cpp/releases/download/v1.9.1/whisper-bin-x64.zip"
Expand-Archive -Path "$base\tools\whisper\whisper-bin-x64.zip" -DestinationPath "$base\tools\whisper" -Force
Get-ChildItem "$base\tools\whisper" -Recurse -Filter whisper-cli.exe | Select-Object -ExpandProperty FullName
```
- 実行ファイルは **`$base\tools\whisper\Release\whisper-cli.exe`** に展開される（`Release` サブフォルダ内。実機で確認済み）。
- **PowerShellでは `curl` は別コマンドの別名**のため、必ず `curl.exe` と書く。

### 3.2 モデルを取得

初期採用モデルは `ggml-medium.bin`（`音声テキスト化方式選定.md` 準拠。精度重視・日本語対応・約1.5GB）。
```powershell
$base = "C:\minutes"
curl.exe -L -o "$base\tools\whisper\models\ggml-medium.bin" "https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-medium.bin"
```
- 処理を軽くしたい場合は `ggml-small.bin`、精度を上げたい場合は `ggml-large-v3.bin` に置き換え可（同じ `models\` に配置）。

### 3.3 動作確認

```powershell
& "C:\minutes\tools\whisper\Release\whisper-cli.exe" --help
```
→ 使い方（オプション一覧）が表示されれば実行可能（実機で確認済み。usage行のパスが文字化けしないこと＝ASCIIパスの確認も兼ねる）。

**根拠**：
- 公式リポジトリ `ggml-org/whisper.cpp`。最新リリース v1.9.1 の `whisper-bin-x64.zip` に `whisper-cli.exe` が含まれることをGitHub APIで確認。
- モデルは HuggingFace `ggerganov/whisper.cpp` で配布。`medium` は初期採用モデル。

**うまくいかない場合**：
- `whisper-cli.exe` が起動時にDLLエラー → MSVC再頒布可能パッケージ（x64）を導入：`winget install --id Microsoft.VCRedist.2015+.x64 --source winget`。
- モデル取得が途中で止まる → `curl.exe -L` を再実行、またはブラウザで直接ダウンロードして `models\` に置く。

## 4. 導入完了チェック

```powershell
$base = "C:\minutes"
ffmpeg -version
ffprobe -version
& "$base\tools\whisper\Release\whisper-cli.exe" --help
Test-Path "$base\tools\whisper\models\ggml-medium.bin"   # True になること
```

- [x] `ffmpeg -version` が表示される（8.1.2）
- [x] `ffprobe -version` が表示される（8.1.2）
- [x] `whisper-cli.exe --help` が表示される（v1.9.1・ASCIIパスで文字化けなし）
- [x] `ggml-medium.bin` が存在する（約1.42GB・取得確認済み）
- [x] `C:\minutes\work` フォルダが存在する

完了したら `手順動作確認.md` の「0. 事前準備の確認」に結果を記録する。

## 付録：調査根拠・実機確認メモ

| 対象 | 根拠・結果 |
|---|---|
| FFmpeg | winget `Gyan.FFmpeg 8.1.2` を導入し `ffmpeg/ffprobe -version` 動作確認。※このビルドは `--enable-whisper` 付き（将来、FFmpeg単体での文字起こしに簡素化できる可能性あり） |
| whisper.cpp | 公式 v1.9.1 `whisper-bin-x64.zip` を展開、`Release\whisper-cli.exe --help` 動作確認 |

| モデル入手先 | HuggingFace `ggerganov/whisper.cpp`（`ggml-medium.bin`） |
| curl | PowerShellでは `curl` は別名のため `curl.exe` を使用 |
