# 議事録自動化 — はじめての方へ（クイックスタート）

このツールは、**Teams等の録画から議事録を作り、Backlogに登録する**までを支援します。
前半（環境構築〜文字起こし）は**スクリプトで自動**、後半（議事録の仕上げ〜登録）は**手作業**です。

> 手順の詳細は `議事録作成手順書.md`、環境の詳細は `環境構築手順書.md` を参照してください。本書はまず動かすための最短ガイドです。

---

## 0. 前提（これが無いと途中で詰まります）

### PC・環境
- **Windows 10 / 11**
- **ソフトのインストール権限**（会社PCで制限されていると FFmpeg 導入が失敗します）
- **winget が使える**こと（`winget -v` が表示される）
- **インターネット接続**（初回に whisper 用モデル 約1.5GB をダウンロード。プロキシ/ファイアウォールで github・huggingface が塞がれていると失敗します）

### アクセス権
- 対象の**録画にアクセスできる**こと（Teams会議の参加者/開催者、OneDrive/SharePoint権限）
- **Backlog 全社共通（ALL）ドキュメントへの登録権限**
- **承認済みのAI**が使えること（※外部AI・外部サーバーを使う場合は、事前に**上長の承認**が必要）

---

## 1. 環境構築（初回だけ・数分）

プロジェクト一式（この README や `setup.ps1` が入ったフォルダ）を入手し、そのフォルダで PowerShell を開いて実行します。

```powershell
powershell -ExecutionPolicy Bypass -File ".\setup.ps1"
```
（フォルダ内で Shift+右クリック →「PowerShell ウィンドウをここで開く」を使うと簡単です）

- FFmpeg・whisper.cpp・モデル・作業フォルダ **`C:\minutes`** を自動で用意します。
- 最後の **Verify が全部 OK** なら完了です。
- `ffmpeg : NOT FOUND (open a new PowerShell...)` と出たら、**新しい PowerShell を開いてもう一度実行**してください（今度は全部 skip/OK になります）。

> 何度実行しても安全です（できている物はスキップ）。途中でダウンロードが失敗しても、もう一度実行すれば続きから取得します。

---

## 2. 録画を用意する

- 議事録にしたい**録画（.mp4）を `C:\minutes\work` に置く**。
- 録画は社内情報です。**外部同期される場所（クラウド共有など）には置かない**でください。

---

## 3. 文字起こし（自動）

```powershell
powershell -ExecutionPolicy Bypass -File "C:\minutes\transcribe.ps1"
```
- `work` 内の録画を自動で選びます（**複数あるときは番号で選択**）。
- 音声抽出 → 文字起こし → 結果 `C:\minutes\work\transcript.txt` を表示します。
- 所要時間は**録画時間の約1倍前後**（例：45分の録画で約40分）。

> **発言者名のヒント**：`C:\minutes\work\names.txt`（UTF-8）に参加者名を書いておくと、文字起こしの名前精度が上がります。**このツールには現在の参加者名を入れた `names.txt` を同梱**しているため、`setup.ps1` を実行すると `C:\minutes\work` に配置されます（既に置いてあれば上書きしません）。参加者が変わったら書き換えてください。ただし日本語のヒントは環境により効きが変わるため、**最終的な名前は Teams のトランスクリプト等で必ず確認**してください。

---

## 4. AIで議事録案を作る（手作業）

1. `議事録作成手順書.md` の「**Step 4**」にあるプロンプトを開く。
2. 先頭の **会議名・開催日**を記入し、末尾に `transcript.txt` の中身を貼り付ける。
3. **承認済みAI**に送信する（外部AIは上長承認が前提）。
4. 出力された議事録（Markdown）を **`YYYYMMDD_会議名_議事録.md`** として保存（**UTF-8・BOMなし・LF**）。

---

## 5. 整形と公開前確認（手作業／**判断が必要**）

- **整形**：9章立て・表形式・タイトル命名を確認（`Markdownフォーマット定義.md`）。
- **公開前確認**：全社公開して問題ないかを確認します。
  - 個人情報・機密情報が無いか
  - **個人の異動・評価・労務などの機微情報**を本文に断定していないか（該当は「要確認」に回す／抽象化）
  - 未確定事項を決定事項として書いていないか

> ここは自動化できない**人の判断**が必要な工程です。**公開可否を判断できる人**が行ってください。

---

## 6. Backlogに登録（手作業）

1. Backlog →「全社共通（ALL）」→ ドキュメント → **登録先の親ドキュメント**を開く。
2. 同名が無いか確認 →「ドキュメントを追加」。
3. タイトルに `YYYYMMDD_会議名_議事録` を入力。
4. 整えた**Markdown本文を貼り付け** → 保存 → 表示崩れがないか確認。

---

## 困ったとき

| 症状 | 対処 |
| --- | --- |
| `ffmpeg` が認識されない | 新しい PowerShell を開いて `setup.ps1` を再実行 |
| 録画が複数あって迷う | 実行時に番号で選べます（新しい録画から並びます） |
| 文字起こしが文字化けして見える | `Get-Content "C:\minutes\work\transcript.txt" -Encoding UTF8` で読む（ファイル自体は正しい） |
| ダウンロードが途中で失敗 | もう一度 `setup.ps1`（できている物はスキップ、失敗分だけ再取得） |
| 録画をチャットからDLできない | 権限の問題。開催者にダウンロード可で共有を依頼 |
| フォルダ名を変える／別PCへ移す | **先に実フォルダをリネーム／配置してから** `setup.ps1` を実行（逆にすると再ダウンロードになります） |
